# knowledge-collector

为忆哒学习应用收集学习资料，转换为忆哒批量生成碎片所需的输入格式。

## 核心理念

**没有万能解析器。** 收到数据后，先分析格式特征，再选择合适的方式处理：
- **结构化数据** → 写 Python 解析脚本（快速、可复现）
- **半结构化/复杂数据** → AI 辅助提取 + 脚本转换
- **非结构化数据** → AI 直接提取并生成忆哒格式

## 忆哒批量生成碎片格式

### 格式规范

```
{{字段1}}{{字段2}}{{字段3}}▮{{字段1}}{{字段2}}{{字段3}}▮{{字段1}}{{字段2}}{{字段3}}
```

**规则：**
- 每个字段用 `{{` 和 `}}` 包裹
- 多个知识点用 `▮` 分隔（忆哒分隔符）
- 每个知识点内的字段顺序必须一致
- 字段数量必须与模板字段数量匹配
- 字段值中不能包含 `{{` `}}` `▮`（需提前清理）

### 字段说明

输出字段由忆哒 App 的模板配置决定，不需要用户指定。收集数据时，根据内容类型自动选择合适的字段映射：

| 内容类型 | 推荐字段 | 示例 |
|---|---|---|
| 词汇 | word, meaning | `{{abandon}}{{v. 放弃；抛弃}}` |
| 词汇+音标 | word, phonetic, meaning | `{{abandon}}{{əˈbændən}}{{v. 放弃；抛弃}}` |
| 选择题 | title, content, answer | `{{#1}}{{题目...}}{{答案：A}}` |
| 知识卡片 | question, answer, tags | `{{什么是光合作用}}{{...}}{{生物}}` |

> ⚠️ 字段数量和顺序需与忆哒 App 模板一致。如不确定模板字段，可询问用户或查看忆哒 App 设置。

## 工作流程

### Step 1: 需求确认（必须）

**收到收集请求后，不要直接开始收集。先通过逐步提问明确用户需求。**

按以下顺序提问（根据用户已提供的信息跳过已有答案的问题）：

#### 1️⃣ 收集什么？
- 具体学科/领域是什么？（如：英语四级词汇、高中历史、数学公式）
- 需要收集哪方面的内容？（如：单词释义、选择题、概念解释）
- 大概需要多少条？（如：50 个单词、100 道题）

#### 2️⃣ 数据从哪来？
- 是否已有数据文件？（如有，让用户指定路径或粘贴内容）
- 是否需要从网络搜索获取？（如需，确认搜索关键词）
- 是否有特定数据源？（如：指定网站、GitHub 仓库、在线数据集）

#### 3️⃣ 版权确认
- 提示用户该内容的版权风险等级
- 用户确认仅用于个人学习后继续

**提问原则：**
- 每次只问 1-2 个问题，不要一次性抛出所有问题
- 根据用户的回答动态调整后续问题
- 如果用户已在初始请求中明确了某些信息，跳过对应问题
- 当需求足够明确（知道收集什么、从哪来、输出什么字段）时，进入下一步

**需求明确的标准：**
- ✅ 知道收集什么内容（学科、类型、数量）
- ✅ 知道数据来源（文件路径 / 网络URL / 搜索关键词）
- ✅ 用户已确认版权提示

**示例对话流程：**
```
用户: 帮我收集英语词汇

→ 追问 1: 收集什么级别的词汇？大概需要多少条？
用户: 四级高频词汇，500 个左右

→ 追问 2: 数据从哪来？我可以用 GitHub 上的开源四级词汇表，或者从网上搜索。你有偏好吗？
用户: 用开源的吧

→ 确认: 好的，我来从 GitHub 开源词汇表中收集约 500 个四级高频词汇，自动映射为忆哒格式字段。该数据为开放许可资源，风险较低。开始收集？
用户: 好的

→ 开始执行 Step 2
```

### Step 2: 收集数据

**核心原则：想尽一切办法，深入挖掘所有可能的数据来源。不要满足于第一个找到的结果，多源交叉验证，确保数据的完整性和准确性。**

#### 收集策略（按优先级依次尝试）

**1. 开源数据集优先**
- GitHub/GitLab 搜索开源学习资料仓库（搜索关键词：学科名 + dataset/vocabulary/quiz/题库）
- Kaggle、HuggingFace Datasets 等数据集平台
- 公共领域数据（政府公开数据、维基百科、Wiktionary）
- 学术开放数据集（Common Crawl、The Pile 等）

**2. 网络广泛搜索**
- `web_search` 多轮搜索，变换关键词组合：
  - 第一轮：核心关键词（如「英语四级词汇表」）
  - 第二轮：加限定词（如「英语四级词汇表 下载 txt」「英语四级高频词汇 开源」）
  - 第三轮：英文搜索（如 "CET4 vocabulary list github"）
  - 第四轮：搜索特定格式（如「英语四级词汇 csv json 制表符」）
- `web_fetch` 抓取搜索结果页面，提取有效数据
- 如果一个来源数据不全，尝试从多个来源拼接

**3. 结构化数据源**
- 维基百科 / Wiktionary（API 或页面抓取）
- 在线词典 API（有道、金山、Free Dictionary API）
- 考试官网公开的大纲和样题
- 教育类网站的公开资源

**4. 用户提供的数据**
- 用户粘贴的文本
- 用户指定的本地文件路径
- 用户提供的 URL

**5. 深度挖掘（当以上方式都不够时）**
- 分析网页结构，找到隐藏的 API 接口
- 从多个页面/分页中批量抓取
- 解析 PDF/Word 文档中的表格和列表
- 从网页的 JSON-LD、structured data 中提取
- 搜索学术论文的附录或补充材料
- 查找是否有对应的 Anki 共享牌组（可导出为 CSV）

#### 收集要求

- **不轻易放弃**：一个来源找不到就换关键词、换平台、换语言再搜
- **多源验证**：同一数据尽量从 2 个以上来源交叉验证
- **主动补全**：如果某来源数据不完整（如只有单词没有释义），主动从其他来源补充
- **数量达标**：收集数量应达到或超过用户要求的数量
- **质量把关**：过滤掉明显错误、重复、格式混乱的数据

#### 收集工具

| 工具 | 用途 |
|---|---|
| `web_search` | 搜索引擎查询，发现数据源 |
| `web_fetch` | 抓取网页内容，提取文本 |
| `read` | 读取本地文件（TXT/MD/JSON/CSV/PDF/Word） |
| `exec` + Python | 运行脚本批量抓取、解析 API、处理复杂格式 |
| `urllib`/`requests` | 在线数据集直接拉取 |

#### 收集流程

```
确定收集目标
    ↓
搜索开源数据集（GitHub/Kaggle/公共领域）
    ↓
找到？──是──→ 下载并验证质量
    │              ↓
    │           数量足够？──是──→ 进入 Step 3
    │              ↓ 否
    │           补充搜索其他来源
    │
    ↓ 否
网络广泛搜索（多轮关键词变换）
    ↓
找到？──是──→ 抓取并验证质量
    │              ↓
    │           数量足够？──是──→ 进入 Step 3
    │              ↓ 否
    │           继续搜索更多来源
    │
    ↓ 否
深度挖掘（API/分页/PDF/Anki等）
    ↓
最终收集结果 → 进入 Step 3
```

### Step 3: 分析格式 → 决策树

收到数据后，按以下决策树选择处理方式：

```
数据格式是否高度结构化？
├─ 是（有明确分隔符/固定格式）
│   ├─ 分隔符格式（制表符/破折号/冒号/竖线）
│   │   → 用 parse_delimited() 或写自定义脚本
│   ├─ 缩进层级格式
│   │   → 用 parse_indented()
│   ├─ 编号列表格式
│   │   → 用 parse_numbered_list()
│   ├─ JSON/CSV 格式
│   │   → 直接用 Python json/csv 模块解析
│   └─ HTML 表格
│       → 用 BeautifulSoup 或正则提取
│
├─ 半结构化（有规律但不完全一致）
│   → AI 辅助提取关键信息 → 脚本转换
│
└─ 非结构化（自由文本、段落、文章）
    → AI 直接提取知识点 → 生成忆哒格式
```

**快速识别技巧：**
- 看前 10 行：是否有统一分隔符？
- 看缩进：是否有层级结构？
- 看编号：是否有 1. 2. 3. 前缀？
- 看编码：是否是 JSON/CSV/HTML？

### Step 4: 处理数据

#### 方式 A：脚本解析（推荐，可复现）

利用 `{baseDir}/scripts/` 下的共享工具模块：

```python
import sys, os
sys.path.insert(0, "{baseDir}/scripts")
from yida_utils import to_yida, validate, read_input, write_output

def parse(content):
    # 根据实际格式写解析逻辑
    items = []
    for line in content.strip().splitlines():
        # ... 提取字段
        items.append({"field1": "...", "field2": "..."})
    return items

content = read_input("input.txt")
items = parse(content)
yida = to_yida(items, ["field1", "field2"])

# 验证格式
ok, errs = validate(yida)
if not ok:
    for e in errs:
        print(f"⚠️ {e}")

write_output(yida, "output.txt")
```

**yida_utils.py 提供的工具：**

| 函数 | 说明 |
|---|---|
| `to_yida(items, field_names)` | 将 dict 列表转为忆哒格式 |
| `validate(yida_str, expected_fields)` | 验证格式正确性 |
| `read_input(path)` | 读取文件（支持路径或 stdin） |
| `write_output(content, path)` | 输出到文件 + 打印预览 |
| `parse_delimited(content, sep)` | 通用分隔符解析 |
| `parse_numbered_list(content)` | 编号列表解析 |
| `parse_key_value(content)` | 键值对解析 |
| `parse_indented(content)` | 缩进层级解析 |

#### 方式 B：AI 辅助（复杂格式）

当数据格式复杂、不规则，或需要语义理解时：

1. **先用 AI 提取结构化信息**
   - 将原始数据交给 AI，要求提取为 JSON 格式
   - 明确指定字段名和提取规则

2. **再用脚本转换为忆哒格式**
   - AI 输出 JSON → `to_yida()` 转换

**示例 prompt：**
```
请从以下内容中提取知识点，输出 JSON 数组：
[{"word": "单词", "meaning": "释义", "example": "例句"}]

要求：
- 每个知识点一个对象
- 字段一致
- 去除无关内容

原始内容：
...
```

#### 方式 C：AI 直接生成（非结构化）

对于文章、段落等自由文本，直接让 AI 生成忆哒格式：

```
请从以下内容中提取关键知识点，直接输出忆哒格式：
{{字段1}}{{字段2}}▮{{字段1}}{{字段2}}▮...

规则：
- 每个知识点 2-3 个字段
- 字段值简洁准确
- 用 ▮ 分隔

原始内容：
...
```

### Step 5: 验证输出

**必须执行的验证清单：**

- [ ] 字段数一致（每个知识点的字段数量相同）
- [ ] 括号匹配（每个 `{{` 都有对应 `}}`）
- [ ] 无空字段（字段值不为空）
- [ ] 无非法字符（字段值中无 `{{` `}}` `▮`）
- [ ] 分隔符正确（知识点间用 `▮` 分隔）
- [ ] 内容准确（抽样检查几条，确保解析正确）

可用 `validate()` 函数自动检查前 3 项。

### Step 6: 输出

**输出方式：**

| 方式 | 说明 |
|---|---|
| 单文件 | 所有结果写入一个 `.txt` 文件 |
| 多文件 | 按类别/分组写入多个文件 |
| 直接打印 | 小量数据直接输出到 stdout |

**命名规范：**
- 单文件：`{主题}_忆哒格式.txt`
- 多文件：`{主题}_{类别}.txt`
- 示例：`英语四级词汇_忆哒格式.txt`、`高考历史_选择题.txt`

## 版权合规

**所有版权风险等级的内容均可收集，但需根据风险等级提示用户。**

收集任何内容前，先评估版权风险等级，然后向用户发出相应提示。用户确认后继续收集。

### 版权风险等级

| 级别 | 说明 | 提示方式 |
|---|---|---|
| 🟢 低风险 | 公共领域、开放许可、事实性数据 | 无需提示，直接收集 |
| 🟡 中风险 | 网站原创内容、用户生成内容 | 提示用户注意使用条款，建议仅供个人学习 |
| 🔴 高风险 | 整本教材、付费课程、官方题库 | ⚠️ 强烈提示版权风险，明确告知仅供个人学习，不得传播或商用，用户确认后继续 |

### 提示模板

**中风险提示：**
> ⚠️ 该内容可能受版权保护（来源：xxx）。建议仅供个人学习使用，不得用于商业用途或公开传播。是否继续？

**高风险提示：**
> ⚠️ 版权风险提示：该内容属于受版权保护的资料（来源：xxx）。收集后仅供个人学习使用，严禁商业用途和公开传播，可能面临法律风险。是否继续？

### 合规使用建议

- 优先使用开放许可资源（CC0、CC-BY、公共领域）
- 完整记录来源（URL、作者、日期）
- 仅供个人学习使用
- 如需商业使用，请获取授权
